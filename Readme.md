Blog原因
===
****
&nbsp;&nbsp;&nbsp;&nbsp;申请Github也已经好久了，以前主要是实验室程序的保密性，加上自己的惰性，一直没有使用。时至今日，依旧没有很好的搞明白，git的各种资源库的原理与流程，对各种git命令也不是很熟悉。最近难得有时间静下心来去审视自己，回首近两年来浑浑噩噩的日子。一心一意埋着头给实验室项目写代码，找bug；帮助创业的哥们写平台，维护。实验室这边由于每届只招一两个人，一个项目基本上就靠着自己在维护；创业的哥们这边，由于以前搞技术的哥们自己情感出现波动，也逐渐退出，自己就硬着头皮上，成为唯一搞技术的苦逼。这两件事情基本上占据了前两年的所有精力。以至于女朋友都一直嫌弃我冷落了她。男人啊，**难**。

&nbsp;&nbsp;&nbsp;&nbsp;努力终于出点成果，实验室的项目在自己的带领下各个模块终于能在恶劣的环境下稳定高效的工作了`主要是在告诉移动环境下：高铁上`；为哥们写的创业平台也稳步在使用阶段，经过半年的运行名声好不错，但是中国互联网市场就是瞬息万变，大鱼吃小鱼，小鱼吃虾米，而且市场畸形，恶意竞争严重，`我们的创业项目主要是订餐平台，现在美团，饿了么，百度和阿里也相继厮杀进来，最近打得火热，一年前我们就遇见到了这个市场，眼光虽有，但是无奈资金有限，虽然早起步一年，但是终还是抵不过大公司的烧钱。`，扯远了-_-|||。。。

&nbsp;&nbsp;&nbsp;&nbsp;之所以又重新开始使用github，是因为最近实验室项目趋于稳定，`实验室做的就是高铁这种高速移动链路质量又那么差的情形下，实现稳定高速上网`，需要些能够直观的展示给别人看的部分。网上的流量监测软件虽然多，但是适合自己的竟没有。于是自己花了个周末写了个[流量监测软件](https://github.com/guyannanfei25/monitor),自己和很多人一样是开源软件的受益人，因此决定将其开源出来。此后会不断开源出自己写的认为不错有意义的软件。

&nbsp;&nbsp;&nbsp;&nbsp;这个软件虽然小，但是回想自己编写并一步步的完善的过程，还是很有意义，很让人成长的，于是决定一点点的记录下来，给自己也给别人`如果大家有耐心看的话`提个借鉴。之所以以这种简陋的形式，没有自己搭个Blog是因为精力真的有限，而且这种简单的展示也满足自己的需求。

>古人云：得意忘言。有时候重在意义不在于形式，界面虽然花哨但是没有太多实用的东西，华而不实不是我所追求的。有时候Simple is best。即使如此，这么简陋的样子，还是让各路大神贱笑了。
